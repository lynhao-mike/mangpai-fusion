## 五、案例目录结构

每个案例的标准目录：
```
cases/C-2026-001/
├── input.md           输入信息（命盘 + 提问）
├── analysis.md        分析过程（含派别仲裁记录）
├── feedback.md        命主反馈
├── lessons.md         本案学到的教训
└── calibration-report.md  自动校准结果（tools/calibrate.py 产出，v1.1）

reports/C-2026-001-report.md   正式对外报告（templates/report.md 格式）
```

### 强制约束（已在 C-2026-001 验证执行）

- ❌ 不允许只有 cases/ 没有 reports/
- ❌ 不允许只有 reports/ 没有 cases/
- ✅ 二者必须**同一次 commit** 提交 ← C-2026-001 已遵守
- ✅ analysis.md 中**所有断语必须双轨置信度**（★+%）← 已遵守
- ✅ 报告 + 案例 + 索引同步更新 ← 已遵守

```
  cases/C-YYYY-NNN/feedback.md
  cases/C-YYYY-NNN/lessons.md

  → 触发 tools/calibrate.py（v1.1）
  → 自动产出 cases/C-YYYY-NNN/calibration-report.md
  → 更新本 cases-index.md
```

## 七、案例分级

| 级别 | 含义 |
|---|---|
| 🟢 已应验 | 命主反馈应验率 ≥ 70%，可作为新规律晋级证据 |
| 🟡 待应期 | 含未来年份预测，等待时间触发 |

| ⚫ 已退档 | 命主撤回授权 / 数据有误 |

## 八、Case ID 命名规则

```
C-YYYY-NNN

YYYY = 立案年份
NNN  = 当年序号（001 起）

例：
  C-2026-001 = 2026 年第 1 案 ← 当前已立案
  C-2026-002 = 2026 年第 2 案
  C-2027-015 = 2027 年第 15 案
```

私密案例：`C-YYYY-NNN-PRIV`（不公开 push）

## 九、案例升级路径

### candidate → confirmed
- 当某条互补层规律在 ≥ 3 个案例中应验 → 升入共识层

- 自动触发 mapping/ 中条目移到注释段
