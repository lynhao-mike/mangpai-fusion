## 五、案例目录结构与强制约束

每个案例的标准目录：

```text
cases/C-YYYY-NNN-{乾/坤}-{干支}/
├── input.md
├── analysis.md
├── feedback.md
├── lessons.md
└── findings/              # 可选，结构化引擎输出

reports/C-YYYY-NNN-{乾/坤}-{干支}-report.md
```

强制约束：

- 不允许只有 cases/ 没有 reports/。
- 不允许只有 reports/ 没有 cases/。
- 报告标题必须含性别：`# 八字分析报告 · C-YYYY-NNN-{乾/坤}-{干支}`。
- 案例序号、命式段（乾/坤）、目录名、报告名、正文 case_id 必须一致。
- 反馈到位后，报告正文不得继续保留“待补 known_facts”作为当前状态。

## 六、案例分级

| 级别 | 含义 |
|---|---|
| 🟢 已应验/高应验 | 命主反馈应验率较高，可作为新规律晋级证据 |
| 🟡 待应期 | 含未来年份预测，等待时间触发 |
| 🟠 部分应验 | 应验率 50-70% 或存在重大漏判，触发规律 review |
| 🔴 失验 | 应验率 < 50%，触发规则降权或退役 |
| ⚪ 待反馈 | known_facts 空或不足，前向预测置信封顶 |
| ⚫ 已退档 | 命主撤回授权 / 数据有误 |

## 七、Case ID 命名规则

```text
C-YYYY-NNN-{乾/坤}-{年柱}{月柱}{日柱}{时柱}

YYYY = 立案年份
NNN  = 当年序号（001 起）
{乾/坤} = 性别命式段；birth.性别=M 对应乾，birth.性别=F 对应坤
{干支} = 八字四柱连写（8 字）
```

私密案例：`C-YYYY-NNN-{乾/坤}-{干支}-PRIV`（不公开 push）。
