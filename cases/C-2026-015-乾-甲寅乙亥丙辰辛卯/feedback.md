- **建议入库新规则（候选 status=candidate）**：
  - 若命主走纯体制路径 + 化杀生枭/官印相生 → **兑现层级上调 1 档**（L11 → L12）

- **补充交叉验证需求**：在后续命例中观察"做功 2 层 + 体制内"是否稳定兑现 L12，若再次复现 → 候选规则升 confirmed

### CFL-C015-002 · 财富域输出与行业方向输出的耦合性 gate（**引擎工程改进**，非规则纠偏）

**问题定位**：

- 报告同时输出：
  - 互补 4.1：**公门 / 国企首选 90%**
  - 互补 4.2 财富：**中富下 L9 85%**
- 二者隐含矛盾——若 P(体制内) = 0.9，则用市场财富分级输出 L9 在工程上不一致：
  - **体制内薪酬走公务员体系**（按级别 / 工龄 / 地域），与市场财富 7 等不直接映射
  - 当前报告等于**两条独立轨道并行输出**，命主或读者无从判断哪个该信

**重要澄清**：

- ✅ 这**不是**"市场财富分级规则错"——本案命主从未走过非体制路径，规则本身没被证伪
- ✅ 这**不是**"维度错配命中失败"——立案时 known_facts 全空，引擎按 fallback 输出市场财富分级是合理的默认
- ⚠️ 这**是**"引擎输出层耦合 gate 缺失"——当其他维度已强暗示某条路径时，本维度应做条件输出而非独立输出

**v1.4 候选工程改进**：

```
若 P(体制内 | 行业方向) > 0.7：
    财富域输出主框架：权力层级（科/处/厅/部/国）+ 公务员薪酬区间
    财富域输出辅注：市场财富分级仅作 reference（标注 "非体制路径下的对照值"）
若 P(体制内 | 行业方向) < 0.3：
    财富域输出主框架：市场财富分级 7 等
    财富域输出辅注：无
若 0.3 ≤ P(体制内) ≤ 0.7：
    财富域输出双框架并列，明确标注两套各自适用条件
```

### CFL-C015-003 · 应期"事件类型"vs"时间窗"分流（**新增**）

**观察**：2010 庚寅应期"财源/置业"在时间上完美命中，但实际事件是"职级升迁"。

**结论**：
- 应期判定应分为 **time_window**（时间窗）+ **event_type**（事件类型）两层独立评估
- 当前规律将"庚寅财星显象"绑死为"财源/置业"，实际在**体制内案例**中"庚透财"也可解读为"职级象征性收益（晋升、待遇调整）"
