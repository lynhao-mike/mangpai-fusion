## 🔧 功能详解

### 1. 增强版应期时间窗推理

**原有问题**:
- 简单 ±2年固定窗口，忽略大运交接期
- 所有领域（婚姻/事业/健康）使用相同宽度
- 未考虑置信度高低差异

**增强方案**:
```python
from engine.application.time_window_estimator import estimate_enhanced_time_window

enhanced_window = estimate_enhanced_time_window(
    gate_results,
    min_confidence_star=3,  # 只考虑 ★≥3 的 gate
    parsed_input=parsed,     # 提取大运信息
)

# 返回结构：
# - primary: 主窗口（最高置信度年份 ±base_width）
# - secondary: 次要窗口列表（其他高置信度峰值）
# - window_reasons: {年份: 成因标注}（大运交接/连续触发/次要峰值）
# - dayun_transitions: 大运交接年份列表
# - confidence_distribution: {年份: 综合置信度}
```

**领域时间敏感度配置**:
- 婚姻/感情：`base_width=1`（±1年），urgency=high
- 事业/财富：`base_width=2`（±2年），urgency=medium
- 健康/意外：`base_width=0`（±6月），urgency=critical

**大运交接加成**: 峰值年靠近大运交接点（±1年）时，窗口宽度 +1

### 2. 外部化事件语义配置

**原有问题**:
- 硬编码 `_DOMAIN_MEANINGS` 字典在 [`prediction_signals.py`](../engine/application/prediction_signals.py:88)
- 无法版本化、流派差异难以表达
- RL 动态权重调整需修改代码

**增强方案**:
创建 [`theory/prediction_models/event-semantics.yaml`](../theory/prediction_models/event-semantics.yaml)：

```yaml
domains:
  婚姻:
    semantics:
      - meaning: "婚变/离合"
        weight: 0.6         # 默认概率权重
        urgency: high        # 时间敏感度
        schools: [ziping, dtt, mp]  # 支持流派
        notes: "冲克夫妻宫、配偶星被伤"
      # ... 更多语义

school_overrides:
  dtt:
    健康:
      extra_semantics:
        - meaning: "寒暖失调"
          weight: 0.7
          # 滴天髓派特有语义

rl_adjustments:
  婚姻_婚变离合_20260620: 0.72  # RL 动态更新（追加写入）
```

**加载器接口**:
```python
from engine.application.event_semantics_loader import get_domain_meanings

# 获取领域语义列表
meanings = get_domain_meanings("婚姻", school="ziping", include_weights=True)
# 返回：[{'meaning': '婚变/离合', 'weight': 0.72, 'urgency': 'high', ...}, ...]
```

### 3. 批量 RL 回溯训练

**原有问题**:
- [`minimal_learning_loop.py`](../engine/application/minimal_learning_loop.py:1) 只支持单案反馈更新
- 历史积累的 `prediction_feedback.jsonl` 未被批量利用
- 缺少训练报告与权重变化趋势可视化

**增强方案**:
```bash
# 批量训练（从 cases/ 扫描所有 prediction_feedback.jsonl）
python -m tools.batch_rl_backtest \
    --cases-dir cases \
    --validation-ratio 0.2 \
    --snapshot-interval 10 \
    --output-dir META

# 输出：
# - META/rl_backtest_report_{timestamp}.md  # 训练报告
# - META/rl_weight_snapshots_{timestamp}.json  # 权重快照序列
# - theory/prediction_models/event-semantics.yaml  # 更新 RL 权重
```

**训练流程**:
1. 扫描所有 `cases/*/prediction_feedback.jsonl`
2. 按时间戳排序（模拟在线学习）
3. 80% 训练集 + 20% 验证集
4. Phase-B 更新流派权重（`expert-weights.yaml`）
5. 每 N 案保存权重快照
6. 验证集评估准确率
7. 生成训练报告与权重趋势图

### 4. 概率校准层

**原有问题**:
- 三流派融合后的原始概率未经校准
- 系统可能过于自信或过于保守
- 不同领域的预测误差模式不同

**增强方案**:
```python
from engine.application.probability_calibrator import ProbabilityCalibrator

# 从历史反馈训练校准器
calibrator = build_calibrator_from_feedback_logs(
    cases_dir=Path("cases"),
    output_path=Path("engine/calibration_params.json"),
)

# 校准概率
calibrated_prob = calibrator.calibrate(
    raw_probability=0.85,
    domain="婚姻",
    method="temperature",  # 或 "platt"
)
# 如果系统在婚姻领域历史预测过于自信，temperature > 1.0 会平滑分布
```

**校准方法**:
1. **Temperature Scaling**: `p_cal = softmax(logit(p) / T)`
   - `T > 1.0`: 平滑分布（降低自信）
   - `T < 1.0`: 尖锐分布（提高自信）

2. **Platt Scaling**: `p_cal = 1 / (1 + exp(A*logit(p) + B))`
   - 用 logistic 回归拟合原始概率到真实结果
   - 需要足够样本（默认 ≥10）

**领域自适应**: 每个领域独立维护校准参数（`temperature`, `platt_a`, `platt_b`）

## 📊 验证案例

### 案例 1: 增强时间窗 vs 遗留版

**命盘**: 乾命，癸酉乙卯戊戌甲寅，事业领域

**遗留版输出**:
```json
{
  "start_year": 2025,
  "end_year": 2028,
  "peak_year": 2026
}
```
**原因**: 简单取最高层数 gate 年份 2026，固定 ±2年

**增强版输出**:
```json
{
  "primary": {
    "start_year": 2025,
    "end_year": 2029,  // 宽度 +1（靠近大运交接）
    "peak_year": 2026
  },
  "window_reasons": {
    "2026": "peak_confidence",
    "2027": "dayun_transition",  // 大运交接年
    "2028": "consecutive_trigger"
  },
  "dayun_transitions": [2027],
  "confidence_distribution": {
    "2025": 0.65,
    "2026": 0.92,  // 最高
    "2027": 0.78,
    "2028": 0.71
  }
}
```
**原因**: 检测到 2027 年为大运交接点，窗口扩展到 2029；标注每年成因

## ⚠️ 注意事项与限制

### 1. 降级策略
所有增强模块都有降级路径：
- 配置文件缺失 → 使用硬编码默认值
- 校准器未训练 → 使用恒等映射（不校准）
- 增强时间窗失败 → 回退到遗留版本

### 2. 样本不足问题
- 概率校准需要 ≥10 个样本才启用领域专用校准
- 样本不足时使用全局默认温度（`global_temperature=1.0`）

### 3. 大运信息缺失
- 如果 `parsed_input` 未包含起运信息，大运交接检测会降级
- 不影响其他增强功能

### 4. 性能影响
- 增强时间窗推理增加 ~5-10ms（单案例）
- 概率校准增加 ~1-2ms（每个事件候选）
- 总体性能影响 <5%
