# 共识层 · CONSENSUS · 4 派全认可铁律

> 金字塔**底层**。本文件登记**4 派全部支持**的铁律。  
> 这一层是命理判断的"无争地带"——任何派别都不会反对。

---

## 编号规则

```
CON-{topic}-{NNN}

例：CON-LF-001 = 共识层·理法·第 1 条
   CON-HUNYIN-003 = 共识层·婚姻·第 3 条
```

每条共识规律必须给出 4 派各自的对应规律 ID。

## 六、共识层维护规则

1. **入选门槛**：必须 4 派均有对应规律支持（同向，可表述不同）
2. **新增流程**：在 4 派 index.yaml 中找到 4 个对应 ID → 写入本文件 → 更新 4 个 yaml 的 `cross_school` + `layer: consensus`
3. **降级**：如发现某派实际持反对意见 → 移入 mapping/conflicts.md 仲裁
4. **优先级**：在分析推断中，共识规律**总是优先于**互补层和独门层
5. **置信度**：默认 ★★★★★ (90%+)，仅当历史命中率显著低于 80% 才降级
