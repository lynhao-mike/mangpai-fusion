# 输入 Schema 契约 · 01-input-schema

> **本文规定 v1.2 起 `cases/C-XXX/input.md` 的严格 schema。**
> preflight.py 按本契约逐字段校验，不通过直接拒绝立案。

最后更新：2026-05-23（W1.2）
版本：v1.2.0
依赖：A 决策（排盘+大运+神煞全部外部化，靠问真 APP）

---

## 一、设计原则

1. **零算法保证**：本系统不实现排盘/大运/神煞算法。所有这些字段都必须从问真 APP（或同等可信来源）已经计算好的结果原样录入。
2. **结构化优先于自由文本**：四柱、大运、神煞必须用 YAML 结构化字段；命主背景、提问可用自由文本。
3. **可机器解析**：所有 D1/D2/D3/D4 引擎从 schema 字段读取，绝不正则提取自由文本。
4. **失败显式**：缺字段/格式错 = preflight 拒绝并指出哪个字段错。

## 三、字段详细规范

### 3.1 schema_version
固定字符串。当前必须为 `1.2.0`。preflight 据此选择 schema 校验器版本。

### 3.2 case_meta
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| case_id | string | ✅ | 必须匹配 `^C-\d{4}-\d{3}-[甲乙丙丁戊己庚辛壬癸][子丑寅卯辰巳午未申酉戌亥]{1,4}` 共 8 个汉字 |
| 立案日期 | date YYYY-MM-DD | ✅ | 不晚于今天 |
| 命主代号 | string | ✅ | 不能含真实姓名（PII 禁止） |
| 策略 | enum | ✅ | A / B / C 之一 |
| 来源 | string | ❌ | 自由 |

### 3.3 birth
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| 性别 | enum | ✅ | M / F |
| 公历 | datetime | ✅ | 精确到分钟 |
| 农历 | string | ❌ | 自由（仅参考） |
| 出生地 | string | ✅ | 至少省+市 |
| 真太阳时校正 | bool | ✅ | true=问真已校正 / false=未校正（preflight 给警告） |

### 3.4 四柱
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| 年柱/月柱/日柱/时柱 | string 2字 | ✅ | 第1字∈天干10，第2字∈地支12，组合∈60甲子 |

**preflight 反推校验**：用 birth.公历 反推四柱（用 lunardate / cnlunar 库），若不一致 → 警告（用户可能用了真太阳时调整后的四柱，允许通过但记 warning）。


### 3.5 藏干
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| 年支/月支/日支/时支 | list[Canggan] | ✅ | 主气必填，中气余气可缺 |

`Canggan` 结构：
```yaml
{干: <天干>, 类型: <主气|中气|余气>, 力量: <0.0-1.0>}
```

**力量参考表**（preflight 给默认值，引擎可覆盖）：
- 主气：1.0
- 中气：0.3（季节末土的中气可上调到 0.4）
- 余气：0.2

### 3.6 大运
| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| 起运岁 | float | ✅ | 0 < x < 12 |
| 起运年 | int | ✅ | = birth.年 + floor(起运岁) |
| 顺逆 | enum | ✅ | 顺 / 逆 |
| 排布 | list[DayunStep] | ✅ | 至少 8 步（覆盖到 80+ 岁） |

`DayunStep` 结构：
```yaml
{序号: 1-12, 干支: 2字, 起讫: "YYYY-YYYY", 起岁: int, 止岁: int}
```

**preflight 一致性校验**：
- 序号连续不跳号
- 干支必须按 60 甲子顺/逆推
- 起讫年差 = 10
- 第 N 步起岁 = 第 N-1 步止岁 + 1

### 3.7 神煞
按"挂在哪个柱"分组录入。**问真 APP 的输出格式**直接复制即可。

**preflight 不校验神煞内容的命理学正确性**——这是"信任问真 APP"原则的体现。

### 3.8 十二长生
日干在 4 支的状态。问真 APP 直接给。preflight 校验：
- 日干必须 = 四柱.日柱[0]
- 4 个状态值必须 ∈ `[长生, 沐浴, 冠带, 临官, 帝旺, 衰, 病, 死, 墓, 绝, 胎, 养]`


### 3.9 known_facts
**最关键的字段。** 这是命主已发生的事实，引擎用于：
- D2 画面合拍：约束输出不得违背事实（如已知 23 岁结婚，杨派不能再说"晚婚"）
- D3 应期校验：已应期的可作为命中数据（feedback_loop 会把这些写入 hits）
- 自迭代：所有 known_facts 在生成报告时会被引擎回测一遍，命中/失验**自动**记录

| 字段 | 类型 | 必填 |
|---|---|---|
| 年份 | int | 时间型事件必填（如结婚、提拔、灾害） |
| 类型 | enum | ✅ 见下表 |
| 事件 | string | 时间型事件必填 |
| 内容 | string | 状态型事件必填（如学历、职业） |

**类型枚举**：
```
婚姻 / 学历 / 职业 / 财运 / 六亲 / 健康 / 子女 / 灾厄 / 牢狱 / 出国 / 其他
```

### 3.10 提问
列表，命主主诉。**策略选择 A 时必填**，B/C 可空。

## 四、preflight.py 完整校验清单

执行顺序（任何一步失败 = 拒绝立案）：

1. **schema_version** 检查（必须 1.2.0）
2. **YAML 解析**（必须能正确解析所有 ```yaml``` 块）
3. **必填字段** 全部存在
4. **case_id 格式** + 干支合法性
5. **指纹防重**（fingerprint = MD5(性别|公历)[:12] 不在 cases-index）
6. **四柱 ↔ 公历 一致性**（反推校验，不一致给 warning）
7. **大运** 内部一致性（顺逆+起岁+干支序列）
8. **藏干** 力量和合理性
9. **十二长生** 日干一致 + 状态值合法
10. **known_facts** 类型枚举合法
11. **目录名 = case_id** 一致

输出形式：
```
[OK] preflight passed: C-2026-001-庚申戊寅壬子辛丑
[WARN] 真太阳时校正=false, 四柱反推与录入一致，假定用户已手动确认
[FAIL] case_id 不合法: 第 8 字 "戍" 非地支（疑似"戌"误打）
```

## 五、问真 APP 字段映射表

> 命理师从问真 APP 录入数据时，按下表对应抄录。

| 问真 APP 屏幕字段 | input.md schema 字段 | 抄录技巧 |
|---|---|---|
| 公历生日 | birth.公历 | 精确到分钟 |
| 农历生日 | birth.农历 | 直接复制 APP 显示 |
| 性别 | birth.性别 | 男→M / 女→F |
| 出生地 | birth.出生地 | 至少省+市 |
| 真太阳时切换 | birth.真太阳时校正 | APP 默认开 → true |
| 命盘四柱 | 四柱.年/月/日/时柱 | 上下顺序为时-日-月-年，**抄入要倒过来** |
| 藏干区（每柱下方）| 藏干.* | 主中余按 APP 顺序录入 |
| 大运表 | 大运.排布 | 起运岁注意是小数 |
| 起运岁数 | 大运.起运岁 | 注意 APP 显示如 "8岁6月22天" → 8.5 |
| 神煞列表 | 神煞 | 按"挂在哪个柱"分组 |
| 十二长生 | 十二长生 | 注意 APP 标注的是日干在各支的状态 |

## 六、最小化输入（仅命主基本档）

如果只想做"画像版"不做应期推算，可省略大量字段。**最小集**：

```yaml
schema_version: 1.2.0
case_meta: {case_id, 立案日期, 命主代号, 策略: B}
birth: {性别, 公历, 真太阳时校正, 出生地}
四柱: {年柱, 月柱, 日柱, 时柱}
藏干: {年支, 月支, 日支, 时支}  # 只填主气
大运: {起运岁, 起运年, 顺逆, 排布: [前 4 步即可]}
神煞: {}  # 可空
十二长生: {日干, 年支, 月支, 日支, 时支}
known_facts: []
提问: []
```

但 D3 应期门会**自动给出"应期数据不全"警告**，三层 gate 会统一降级到最多 ★★★。
