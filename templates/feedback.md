# 反馈采集模板 · v1.0

> 命主反馈是 mangpai-fusion 的命脉。  
> 每次分析后必须按本模板采集反馈，回流到校准引擎。

---

## 使用说明

统一版命理师内容报告输出后 → 命理师与命主对话 → 收集反馈 → 填入本模板 → 触发 `tools/feedback_ingest.py`

## 六、自动化校准

> 反馈写入完成后，自动触发以下操作：

```bash
# 调用当前反馈摄入入口
python -m tools.feedback_ingest C-YYYY-NNN
```

校准引擎会自动：
1. 更新所有涉及规律的 `hit_count` / `miss_count`
2. 重算 `dynamic_score` / `final_score` / `star`
3. 触发升降级判定（candidate ↔ promoted ↔ retired）
4. 写入 `META/rule-changelog.md`
5. 同步更新 `mapping/` 目录
6. 如规律层级变化（如独门 → 互补） → 自动迁移条目
7. 输出报告 `cases/C-YYYY-NNN/calibration-report.md`

## 七、采集反馈的注意事项

### 7.1 客观证据 vs 主观感受

| 类型 | 处理 |
|---|---|
| **客观证据**（医院诊断书、公司公告、政府证书） | 全权重（multiplier = 1.0） |
| **可验证事实**（领证日期、毕业院校、就业公司） | 全权重 |
| **命主自报**（情绪、隐私事件、回忆） | 80% 权重（multiplier = 0.8） |
| **命主推测**（"我觉得是因为..."） | 不计入应验/失验 |

### 7.2 不要诱导式提问

❌ 错误：「你 2018 年是不是结婚了？」（命主可能附和）
✅ 正确：「2018 年对您来说是怎样的一年？请描述一下。」

### 7.3 警惕命主的"附和效应"

如果命主什么都说"对对对"，需要交叉验证：
- 用反向问题验证（"那 2017 年呢？" → 看是否也"对"）
- 用具体细节验证（"具体是哪个月？" → 看是否能给出）

### 7.4 部分应验的处理

如果断语方向对但精度不够（例：预测 2018 应婚，实际 2017 末领证）：
- 记为 `partial`
- 不计入 hit_count，也不计入 miss_count
- 但触发"应期窗口校准" → 后续看是否需调整

## 八、隐私保护

反馈记录可能包含命主敏感信息：
- 涉及健康疾病 / 隐私事件 / 家庭矛盾的反馈 → 标记 `[PRIVATE]`
- `[PRIVATE]` 段不会出现在公开报告 / GitHub push 中
- 仅本地 + 命理师可访问

如命主撤回授权 → 删除 cases/C-YYYY-NNN/ 整个目录 → 校准记录标记 `[REVOKED]`。
