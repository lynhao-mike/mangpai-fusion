> **创建日期**：2026-05-22
> **状态**：active
> **用途**：定义双仓共识候选规律从 candidate → confirmed → production 的晋级流程

---

## 1 · 三级晋级架构

```
┌─────────────────────────────────────────────────────┐
│  Level 3 · PRODUCTION（生产层 · Month 3 启动）       │
│  ├─ 仅吸收 confirmed ≥1 月稳定 + ≥3 案例覆盖        │
│  ├─ 写入 mangpai-production 仓库（待建）             │
│  └─ 编码：MP-PROD-NNN                               │
└─────────────────────────────────────────────────────┘
                  ▲ 晋级条件见 §2.3
                  │
┌─────────────────────────────────────────────────────┐
│  Level 2 · CONFIRMED（确认层 · 本文件管理）           │
│  ├─ 双仓共识 + 已应验 OR ≥3 案例覆盖                 │
│  ├─ 存储位置：shared-rules/confirmed-rules.md        │
│  └─ 编码：CC-YYYY-NNN（保持原 ID）                   │
└─────────────────────────────────────────────────────┘
                  ▲ 晋级条件见 §2.2
                  │
┌─────────────────────────────────────────────────────┐
│  Level 1 · CANDIDATE（候选层）                       │
│  ├─ 双跑对比矩阵中产出的 D-1/D-2 共识               │
│  ├─ 存储位置：shared-rules/consensus-candidates.md   │
│  └─ 编码：CC-YYYY-NNN                               │
└─────────────────────────────────────────────────────┘
```

## 2 · 晋级条件

### 2.1 进入 CANDIDATE（Level 1）

| 条件 | 必须/可选 | 说明 |
|---|---|---|
| 双跑产出 | 必须 | 必须来自双仓双跑对比矩阵的 D-1 或 D-2 区 |
| 双仓来源标注 | 必须 | A 路径规则 ID + B 路径规则 ID 均已标注 |
| CC-ID 分配 | 必须 | 格式 CC-YYYY-NNN，全局唯一递增 |

### 2.2 CANDIDATE → CONFIRMED（Level 1 → Level 2）

满足以下**任一路径**即可晋级：

| 路径 | 条件 | 适用场景 |
|---|---|---|
| **路径 A：多案例验证** | ≥3 个不同案例双跑中被独立验证 | 通用路径 |
| **路径 B：命主精确反馈** | 有命主一手确认（精确到日期+事实）| META-DR-9 加速晋级 |
| **路径 C：铁律级共识** | 双仓 ★★★★★ + 理论依据无争议 + ≥2 案例 | 快速通道 |

**否决条件**（任一满足则不得晋级）：
- 存在 D-4 冲突未仲裁
- 单仓置信度 < 0.6/2.4
- 仅单一案例支撑且无命主反馈

### 2.3 CONFIRMED → PRODUCTION（Level 2 → Level 3）

| 条件 | 必须/可选 | 说明 |
|---|---|---|
| confirmed 状态 ≥1 个月 | 必须 | 防止新鲜偏差 |
| ≥3 不同案例覆盖 | 必须 | 确保泛化性 |
| 无后续证伪 | 必须 | confirmed 期间无反面证据 |
| mangpai-production 仓库已建 | 必须 | Month 3 启动后才开放 |
| 精简表述（≤50 字规则正文） | 必须 | 生产层追求极简 |

## 3 · 回退机制

### 3.1 CONFIRMED → CANDIDATE（降级）

| 触发条件 | 处理 |
|---|---|
| 新案例双跑产出 D-4 冲突且该规则被涉及 | 降级 + 开仲裁工单 |
| 命主反馈证伪（精确到事实） | 降级 + 标注证伪案例 |
| 3 个月内无新验证且仅单案例支撑 | 标记"休眠"（不删除但不计入统计）|

### 3.2 PRODUCTION → CONFIRMED（降级）

| 触发条件 | 处理 |
|---|---|
| 生产环境中产出错误预测 ≥2 次 | 降级回 confirmed + 重新验证 |
| 理论基础被新学术源材料推翻 | 降级 + 开仲裁工单 |

## 4 · 操作流程

### 4.1 每次双跑后

```
1. 产出对比矩阵 → 标注 D-1/D-2 区共识
2. 分配 CC-ID → 写入 consensus-candidates.md
3. 检查已有 candidate 是否被本次新案例验证
4. 满足晋级条件的 → 移入 confirmed-rules.md
5. 更新统计数据
```

### 4.2 月度健康检查

```
1. 检查所有 confirmed 规则是否有新证伪
2. 检查"休眠"候选是否有新验证可激活
3. 检查 production 晋级条件（Month 3 起）
4. 输出月度报告
```
