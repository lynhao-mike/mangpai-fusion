# 反馈校准下的权重修正规则草案（审查态）

> 目标：在不破坏派系隔离和裁判层契约的前提下，根据真实反馈对领域权重进行保守修正。
>
> 状态：审查态草案；不写入 YAML；不改生产代码；不作为最终参数。

## 1. 修正规则总原则

权重修正必须满足以下原则：

1. **先验优先**：没有足够反馈时，以初始先验为主。
2. **小步调整**：单轮修正幅度要受限，避免权重剧烈漂移。
3. **域内修正**：只修正“某流派在某功能域”的权重，不跨域扩散。
4. **证据优先**：高质量反馈比单纯命中数更重要。
5. **低样本保守**：样本不足时只允许微调或不调。
6. **可追踪**：每次修正必须记录原因、样本量、时间点。

## 2. 权重修正公式草案

建议采用“先验权重 × 修正系数”的形式：

```text
new_weight = prior_weight × correction_factor
```

其中：

```text
correction_factor = 1 + delta
```

`delta` 由以下因素组合而成：

| 因素 | 说明 | 影响方向 |
|---|---|---|
| hit_rate_delta | 该派在该域命中率相对基线的偏差 | 命中高则上调 |
| confidence_delta | 结论置信度与证据质量综合表现 | 越稳越可上调 |
| sample_delta | 样本量修正 | 样本少则收缩 |
| conflict_delta | 冲突率修正 | 冲突多则下调 |
| stability_delta | 前后连续轮次稳定性 | 稳定则上调 |

建议最终 `delta` 控制在较小范围内，例如 `[-0.05, +0.05]` 或更保守。

## 3. 每轮修正上限

建议设置以下硬限制：

1. **单轮最大变动**：单个流派在单个功能域中的权重变动不超过 5 个百分点。
2. **极低样本门槛**：有效样本数不足时，单轮最大变动不超过 2 个百分点，甚至直接冻结。
3. **总和约束**：修正后各派权重总和仍然必须为 100%。
4. **回退机制**：若修正后表现不升反降，可恢复上一版先验。

## 4. 样本门槛建议

| 有效样本数 N_eff | 允许动作 |
|---|---|
| 0-10 | 不建议调权，保持冻结 |
| 11-30 | 允许轻微微调，单轮不超过 2% |
| 31-80 | 允许保守调权，单轮不超过 5% |
| 81-200 | 可进行较稳健修正，但仍需审查 |
| 200+ | 可考虑更系统的权重更新，但仍保留人工复核 |

这里的 `N_eff` 应来自真实有效反馈，而不是原始案例总数。

## 5. 权重漂移触发条件

只有满足以下条件之一，才考虑触发权重漂移审查：

1. 某派在某域连续多轮显著优于/劣于先验预期；
2. 某域样本量达到门槛且趋势稳定；
3. 某派在某域的冲突率持续升高或持续降低；
4. 新增反馈显著改变 Beta 均值或 Wilson 区间；
5. 人工审查认为原先验明显偏离真实表现。

## 6. 权重漂移的安全阈值

为了防止误判导致权重暴走，建议增加以下阈值：

- 单轮修正不得超过上限；
- 累积修正不得突破预设安全边界，例如 ±10%（相对先验）；
- 若连续两轮修正方向相反，说明数据不稳定，应暂停调权；
- 若样本置信区间过宽，则暂停修正。

## 7. 与裁判模型契约的衔接

本修正规则只影响裁判层里的 `prior_domain_weight` 与 `feedback_weight`，不影响：

- 各流派内部规则库；
- 各流派输出结构；
- 统一判断轴定义；
- 派系隔离边界。

衔接方式建议为：

1. 裁判层读取当前领域权重矩阵；
2. 结合新反馈生成修正建议；
3. 经人工审查后更新审查态权重表；
4. 若后续进入结构化规则库，再由规则层读取最新版本。

## 8. 审查结论标准

一版权重修正建议要通过审查，至少要满足：

1. 样本量达到门槛；
2. 方向性足够明确；
3. 不是单次偶然波动；
4. 变动幅度在安全阈值内；
5. 能解释为何上调或下调；
6. 不会破坏派系隔离与裁判层契约。

## 9. 小结

反馈校准下的权重修正应当是“慢变化、可回退、可审查”的过程。当前最重要的不是追求快速收敛，而是避免小样本误导和跨派污染。
